## 2. scope 定義

| scope | 略記 | 対象 |
|---|---|---|
| `architecture` | `arch` | システム構成、データフロー、レイヤー境界、DB スキーマ |
| `spec` | `spec` | 機能仕様、データモデル、API 契約 |
| `ui` | `ui` | ビジュアルデザイン、レイアウト、インタラクション |
| `policy` | `pol` | 開発プロセス、コーディング規約、リリース方針 |
| `product` | `prod` | プロダクト方針、命名、対象ユーザー、収益モデル |
| `performance` | `perf` | パフォーマンス目標、最適化方針、計測戦略 |
| `meta` | `meta` | GDR 運用ルール自体の変更、ドキュメント構造 |
| | | {プロジェクト固有の scope を追加} |

## 3. PREFIX 定義

| PREFIX | ドメイン | 代表的な scope | 用途 |
|---|---|---|---|
| INFRA | インフラ基盤 | `arch`, `pol` | エラー処理、リトライ戦略、共通基盤 |
| PERF | パフォーマンス | `perf`, `arch` | 応答時間目標、最適化方針、計測戦略 |
| LOG | ログ・監査 | `arch`, `pol` | ログ永続化、ローテーション、監査証跡 |
| APP | アプリ・プロダクト識別 | `prod` | 識別子、命名、バージョニング |
| META | メタ（GDR 運用） | `meta` | GDR 運用ルール自体の変更 |
| | | | {プロジェクト固有の PREFIX を追加} |
