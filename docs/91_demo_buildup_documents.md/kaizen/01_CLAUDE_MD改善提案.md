## 1. GDR（General Decision Record）

**GDR-META-001: CLAUDE.md にディレクトリ構成を明記する**

- **status:** Implemented
- **scope:** meta
- **決定:** `documents/01_GDR/markdown/` 配下の構造（仕様・ガイド・テンプレート・サンプル）と `docs/`（Docsify サイト）の役割を CLAUDE.md に記載する
- **理由:** ファイル配置が暗黙知になっており、AI が新規ファイルを誤った場所に作成するリスクがある。ディレクトリ構成を明示することで正確なファイル操作が可能になる
- **影響:** CLAUDE.md にディレクトリツリーを追記する
- **再検討条件:** ディレクトリ構成が大幅に変更された場合

**GDR-META-002: CLAUDE.md に GDR 書式ルールを明記する**

- **status:** Implemented
- **scope:** meta
- **決定:** GDR レコードの必須フィールド（`status`, `scope`, `決定`, `理由`, `影響`, `再検討条件`）を CLAUDE.md に記載する
- **理由:** 現状では AI が GDR を作成するたびに 01_GENERAL_DECISION_RECORD.md を参照する必要がある。CLAUDE.md に書式ルールがあれば `scope` 抜け等のフィールド欠落を防止でき、参照コストも削減できる
- **影響:** CLAUDE.md に書式テンプレートを追記する
- **再検討条件:** GDR 書式自体が改訂された場合

**GDR-META-003: CLAUDE.md に採番ルールを明記する**

- **status:** Implemented
- **scope:** meta
- **決定:** ファイル名のプレフィックス（`01_`, `02_`, `T1_`）、GDR レコード ID（`GDR-{PREFIX}-{番号}`）、サブセクション番号（`001.1`）の採番規則を CLAUDE.md に記載する。リネーム時は見出し・相互リンク・README・`_sidebar.md` を連動更新する旨も含める
- **理由:** 採番の不整合が発生した場合、手動での修正箇所が多くなり漏れやすい。ルールを明示することで採番変更時の影響範囲を AI が自律的に判断できる
- **影響:** CLAUDE.md に採番規則セクションを追記する
- **再検討条件:** 採番体系を変更する場合

**GDR-META-004: CLAUDE.md に Docsify サイドバー同期ルールを明記する**

- **status:** Implemented
- **scope:** meta
- **決定:** markdown ファイルの追加・削除・リネーム時に `documents/01_GDR/markdown/_sidebar.md` も更新する旨を CLAUDE.md に記載する
- **理由:** サイドバーの更新漏れが発生すると、ブラウザでのドキュメント閲覧時にリンク切れや新規文書の欠落が起きる
- **影響:** CLAUDE.md に運用ルールを追記する
- **再検討条件:** Docsify 以外のドキュメントサイトツールに移行した場合

**GDR-META-005: CLAUDE.md にコミットメッセージの言語規則を明記する**

- **status:** Implemented
- **scope:** meta, pol
- **決定:** コミットメッセージは英語で記述する
- **理由:** 既存のコミット履歴が英語で統一されており、一貫性を維持するため
- **影響:** CLAUDE.md に記載するのみ。既存運用の明文化
- **再検討条件:** チームメンバーの言語構成が変わり、日本語の方が適切になった場合

## 2. 現状

1. CLAUDE.md にディレクトリ構成の記載がなく、AI が新規ファイルを誤った場所に作成するリスクがある
2. GDR 書式ルールが CLAUDE.md に記載されておらず、毎回 01_GENERAL_DECISION_RECORD.md を参照する必要がある
3. 採番ルールが暗黙知であり、リネーム時の連動更新箇所が不明確
4. Docsify サイドバーの同期ルールが未記載で、更新漏れによるリンク切れが発生しうる
5. コミットメッセージの言語規則が明文化されていない


---

## 3. 改善案

CLAUDE.md に運用ルール（ディレクトリ構成・GDR 書式・採番・Docsify 同期・コミット規約）を集約し、AI が自律的に正しい判断を行えるようにする。


---

## 4. 詳細セクション

### 4.1. ディレクトリ構成の明示

`documents/01_GDR/markdown/` 配下のツリー（仕様・ガイド・テンプレート）と `docs/`（Docsify サイト）の役割を記載する。

### 4.2. GDR 書式の集約

必須 6 フィールド（`status`, `scope`, `決定`, `理由`, `影響`, `再検討条件`）のテンプレートを CLAUDE.md に直接記載し、参照コストを削減する。

### 4.3. 採番ルールの明文化

ファイル名（`01_`, `T1_`）、GDR ID（`GDR-{PREFIX}-{番号}`）、サブセクション番号、リネーム時の連動更新対象を一箇所に集約する。

### 4.4. Docsify 同期ルール

markdown ファイルの追加・削除・リネーム時に `_sidebar.md` を必ず同期更新する旨を記載する。

### 4.5. コミット規約

既存の英語コミット履歴との一貫性を維持するルールを明文化する。


---

## 5. 検討課題

1. **GDR 書式改訂時の連動更新漏れ**
   - 解決策: GDR 書式を変更した場合は CLAUDE.md の書式テンプレートも連動更新する運用ルールを設ける
   - → フェーズ 1 のタスク 1.2 で CLAUDE.md に書式を追記済み。今後の改訂時は同様に対応する
2. **ディレクトリ変更時のパス追従漏れ**
   - 解決策: CLAUDE.md の採番ルールセクションにリネーム時の連動更新対象を明記する
   - → フェーズ 1 のタスク 1.3 で対応済み
