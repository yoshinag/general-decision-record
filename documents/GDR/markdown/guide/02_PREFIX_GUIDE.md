GDR の `{PREFIX}` はドメインを示す短い識別子である。PREFIX の選定はプロジェクトや業界によって異なるが、以下に共通的なものと業種別の例を示す。

## 1. 共通 PREFIX

業種を問わず多くのプロジェクトで利用される PREFIX。

| PREFIX | ドメイン | 代表的な scope | 用途 |
|---|---|---|---|
| INFRA | インフラ基盤 | `arch`, `pol` | エラー処理、リトライ戦略、共通基盤の設計方針 |
| PERF | パフォーマンス | `perf`, `arch` | 応答時間目標、最適化方針、計測戦略 |
| LOG | ログ・監査 | `arch`, `pol` | ログ永続化、ローテーション、監査証跡 |
| SEC | セキュリティ | `pol`, `arch` | 認証・認可方式、暗号化方針、脆弱性対応 |
| DATA | データ管理 | `arch`, `spec` | データモデル、バックアップ、保持期間 |
| META | メタ | `meta` | GDR 運用ルール自体の変更、ドキュメント構造 |
| APP | アプリ・プロダクト識別 | `prod` | 識別子、命名、バージョニング |

## 13. 導入ガイド

プロジェクトに GDR を導入する際:
1. 本ガイドの共通 PREFIX + 該当業種セクションから PREFIX を選定する
2. 複数業種にまたがるプロジェクトでは各業種から必要な PREFIX を組み合わせる
3. プロジェクト固有の PREFIX が必要であれば追加定義する
4. 使用する PREFIX と scope の対応を Context Definitions（前提定義書）に記載する
5. 新しい PREFIX を追加する場合は `GDR_INDEX` に新セクションを作成する
