## 001. モバイルレイアウトとフォントサイズの改善

### GDR（General Decision Record）

**GDR-UI-001: フォントサイズはモバイル画面向けの値を直接指定する**

- **scope:** ui
- **決定:** シーン定義の `font_size` はモバイル画面（375〜430px幅）で適切に表示される値を直接指定する。`FontScaler`（`InheritedWidget`）はタブレット対応の拡大方向スケーリングとして残すが、`designWidth` をモバイル基準（393px）に設定し、モバイルでは等倍で動作させる
- **理由:** 本アプリはモバイル専用（iOS / Android）。デスクトップ基準（1280px）で設計して縮小する方式は、スケーリング係数の調整が複雑になり、意図しない表示崩れの原因となる。モバイル向けの値を直接指定する方がシンプルで予測可能
- **影響:** 既存のデスクトップ基準フォントサイズ（11〜18px）をモバイル向け（14〜28px）に引き上げる必要がある。`FontScaler.designWidth` を 1280 → 393 に変更する
- **再検討条件:** タブレット最適化やマルチウィンドウ対応が必要になった場合

**GDR-UI-002: flex ウィジェットに direction: "wrap" を追加する**

- **scope:** ui
- **決定:** `FlexWidgetBuilder` に `Wrap` レイアウトを追加し、シーン定義から `direction: "wrap"` で利用可能にする
- **理由:** カテゴリフィルタボタン等の可変個数・可変テキスト長の要素を `Row`（固定幅）や `Grid`（固定アスペクト比）で並べるとモバイルでオーバーフローする。`Wrap` はテキスト長に応じた自然なサイズで自動折り返しするため最適
- **影響:** 既存シーンに影響なし（新しい direction 値の追加のみ）
- **再検討条件:** `Wrap` の spacing / runSpacing をシーンから制御する需要が発生した場合

---

### 現状

1. シーン定義のフォントサイズがデスクトップ基準（1280×720px）の小さい値で設計されており、モバイル画面でテキストが判読困難
2. カテゴリフィルタボタン（`font_size: 11〜13`）が小さすぎて読めない
3. 4シーン中3シーンのコンテナが `width: 1280, height: 720` 固定で、モバイルでオーバーフロー発生
4. カテゴリボタンが `flex row` + 固定幅（140〜170px）で横並びのため画面外にはみ出す
5. `FontScaler.designWidth` が 1280 のまま残っており、GDR-UI-001 の方針（モバイル等倍）と矛盾している

### 理由

モバイル専用アプリケーションとして、画面幅に適したフォントサイズとレスポンシブレイアウトは必須。デスクトップ基準の固定値ではモバイルでの可読性・操作性が著しく低下しており、ユーザー体験を損なっている。

### 改善案

全シーンのレイアウトをモバイル向けに統一する。コンテナは `%` 指定、フォントサイズはモバイルで可読な値を直接指定、カテゴリボタンは `flex wrap` で自動折り返し。`FontScaler` の基準幅をモバイル（393px）に変更し、モバイルで等倍動作させる。

### 実装記録

#### 001.1 FontScaler の導入（実装済み・595373b）

`lib/utils/font_scaler.dart` に `InheritedWidget` として実装。`SceneRendererScreen` の `LayoutBuilder` 内で画面幅からスケール係数を算出し、子ツリー全体に提供する。`designWidth = 393.0`（iPhone 15 基準）でモバイルでは等倍動作。

```dart
static const double designWidth = 393.0;

static double computeScale(double screenWidth) {
  return sqrt(screenWidth / designWidth).clamp(0.9, 1.5);
}
```

スケール値:

| デバイス      | 幅    | スケール | font_size:18→ | font_size:14→ |
|--------------|-------|---------|---------------|---------------|
| iPhone SE    | 375px | 0.98    | 17.6px        | 13.7px        |
| iPhone 15    | 393px | 1.00    | 18.0px        | 14.0px        |
| iPhone Plus  | 430px | 1.05    | 18.8px        | 14.6px        |
| iPad mini    | 744px | 1.38    | 24.8px        | 19.3px        |

適用済みビルダー: `TextWidgetBuilder`, `MarkdownWidgetBuilder`, `CommonButton`, `ContainerWidgetBuilder`

#### 001.2 flex direction: "wrap" の追加（実装済み・595373b）

`FlexWidgetBuilder._buildFlex()` に `direction == 'wrap'` 分岐を追加。`Wrap` ウィジェットにより子要素がテキスト長に応じた自然なサイズで折り返し配置される。

```yaml
- type: flex
  properties:
    direction: "wrap"
    mainAxisAlignment: "start"
    children:
      - type: button
        properties:
          text: "アルカリ金属"
```

#### 001.3 シーンファイルのモバイル対応（実装済み・4b2d868）

| シーン                              | コンテナ        | カテゴリボタン      | ボタンフォント |
|------------------------------------|----------------|-------------------|-------------|
| `periodic_table_simple/start.md`   | 1280×720 固定   | なし               | 18px        |
| `1/start.md` (周期)                | 1280×720 固定   | row + 140px固定    | 13px        |
| `trivia/start.md`                  | 1280×720 固定   | row + 170px固定    | 13px        |
| `flashcard/start.md`               | 1280×720 固定   | なし               | -           |

各シーンに適用した変更:
1. コンテナ: `width: 1280, height: 720` → `width: "100%", height: "100%"`
2. カテゴリボタン: `flex row` + 固定幅 → `flex wrap`
3. フォントサイズをモバイル可読な値に引き上げ（タイトル: 28、ボタン: 18、説明テキスト: 16）
4. 座標: 固定px → `%` 指定

#### 001.4 ButtonWidgetConfig への fontSize 正式追加（実装済み・706ffd7）

`ButtonWidgetConfig` に `fontSize` フィールドがなく `actionProperties` から間接取得していたため、正式なフィールドとして追加。`CommonButton` 側では `FontScaler.scaled(context, config.fontSize)` を適用。

対象ファイル:
1. `lib/models/configs/widget_config.dart` — `ButtonWidgetConfig` に `@Default(24.0) double fontSize` 追加
2. `lib/models/configs/widget_config_parser.dart` — `_button()` に `fontSize` パース追加
3. `lib/factories/builders/button_widget_builder.dart` — `config.fontSize` を直接参照に変更

#### 001.5 Wrap の spacing をシーンから制御可能にする（実装済み・706ffd7）

`FlexWidgetConfig` に `spacing`（double, default 8.0）と `runSpacing`（double, default 8.0）フィールドを追加し、シーン定義から制御可能にした。

#### 001.6 Quiz / MultiChoiceDialog 内部フォントの FontScaler 対応（実装済み・706ffd7）

`quiz_widget.dart` / `multi_choice_dialog_widget.dart` のハードコードフォントサイズを `FontScaler.scaled()` に置き換え。将来的な OS フォントサイズ設定（アクセシビリティ）への対応基盤にもなる。
